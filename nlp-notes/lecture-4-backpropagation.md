## 核心概念

### 反向传播 vs 前向传播

```
前向：输入 → 加权求和 → sigmoid → 输出 → 误差
反向：误差 → 每层局部梯度 → × 上游梯度 → 更新权重
```

反向 = 倒着算。传播 = 误差信号一层一层往回传。

### 完整走一遍

```python
输入:  x = [0.8, 0.9, 0.3]
权重:  w = [0.5, 0.9, 0.1]
z = 1.24, sigmoid(z) = 0.78
真实答案 = 1.0, 误差 = -0.22

# 反向传播：
sigmoid梯度 = 0.78 × (1 - 0.78) = 0.17
误差穿过sigmoid = -0.22 × 0.17 = -0.037

分给"太"(0.8):  -0.037 × 0.8 = -0.030
分给"好看"(0.9): -0.037 × 0.9 = -0.033  ← 最多
分给"了"(0.3):  -0.037 × 0.3 = -0.011

更新：
w_好看 = 0.9 - 0.01 × (-0.033) = 0.90033  ← 略微调大
```

### 核心公式

```
下游梯度 = 上游梯度 × 局部梯度
```

每层只关心自己的局部梯度，乘以上游传下来的信号即可。

## 训练技巧

| 技巧 | 要点 |
|------|------|
| 权重初始化 | 不能全为 0（对称问题），随机小数 |
| ReLU | 正数原样过，负数截断为 0 |
| Adam 优化器 | 自适应学习率，默认首选 |
| L2 正则化 | 防止过拟合（只会背答案不会变通） |

## 为什么叫"自动微分"

PyTorch 里写 `.backward()` 的时候，框架自动走了上面的全过程——画计算图 → 前向算误差 → 反向算梯度 → 更新权重。不需要手算。
